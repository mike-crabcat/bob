Memory Pipeline Browser

Which browsing pattern fits how you'd debug extraction issues?

A

Tabbed Record Types

Top tabs: Bulletins | Claims | Entities | Relations. Each tab shows a filterable list. Click a record to see its detail panel. Cross-link between tabs (e.g. a claim links to its source bulletin and target entity).

B

Pipeline Trace View

Start from a bulletin, see its extracted claims, then the resulting entity changes — a vertical trace. Good for "did this specific conversation produce good memory?" but harder to browse broadly.

C

Both — Tabs for Browsing, Trace for Debugging

Tabbed record browser as the main view, with a "trace" button on any bulletin/claim that opens the chain view. More to build but covers both exploratory browsing and focused debugging.
